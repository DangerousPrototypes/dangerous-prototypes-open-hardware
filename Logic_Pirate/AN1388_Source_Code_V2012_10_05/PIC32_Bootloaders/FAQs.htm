1)      When I compile the bootloader, I get linker errors. What could be wrong?

There could be one of the following possibilities.

 

·         Check the compiler. Make sure that you are using the XC32 compiler.

·         Compiler optimization levels are not supported if you are using free version of XC32 compiler.  In this case you may have to increase the size of the memory region “kseg0_program_mem” in the bootloader linker script.  The demo application’s linker script too need changes so that the application memory region and bootloader memory region does not overlap with each other.

·         Make sure that you have added the correct peripheral library in the linker script file for the selected part number. Check the line OPTIONAL("libmchp_peripheral_32MX795F512L.a")  in the linker script file.  May be the peripheral library name does not match the part number selected.

 

2)      How can I debug bootloader and application together?

 

You can debug bootloader and application together in MPLAB X V1.20. With bootloader set as main project, add the application project under “Loadables”.  Rebiuld the main project and download the code into the target device using a suitable debugger. You should now be able to debug both bootloader and application together.
