Bootloader support

 

The bootloader code is not directly supported on PIC32MX1xx and PIC32MX2xx devices. User will have to make the following modifications to make the bootloader work on PIC32MX1xx and PIC32MX2xx devices.

 

·         On UART bootloader:

o   Remove the existing linker script file from “UART_Btl_Explorer16” workspace and add the example linker script file from the location “Bootloader\linker_scripts\PIC32MX_1XX_2XX\example.ld”

o   In the linker script file, modify the line OPTIONAL("libmchp_peripheral_32MX220F032D.a"), to include the correct peripheral library for the chosen part number.

o   Depending on the end product, user has to map the UART RX and TX ports to required pins using PPS (Peripheral pin select).  This must be done inside the function UartInit() just before configuring the UART registers.

o   User has to design a mechanism suitable to his end product to make the bootloader enter firmware upgrade mode.  This can be done by modifying the macro “ReadSwitchStatus()” in the file HardwareProfile_PIC32MX_PIM_Explorer_16.h.

 

·         On SD Card Bootloader:

o   Remove the existing linker script file from “SD_Card_Btl_Explorer16” workspace and add the example linker script file from the location “Bootloader\linker_scripts\PIC32MX_1XX_2XX\example.ld”

o   In the linker script file, modify the line OPTIONAL("libmchp_peripheral_32MX220F032D.a"), to include the correct peripheral library for the chosen part number.

o   Depending on the end product, user has to map the SPI ports to required pins using PPS (Peripheral pin select).  This must be done inside the function main() in the file sd_bootloader.c.

o   User has to design a mechanism suitable to his end product to make the bootloader enter firmware upgrade mode.  This can be done by modifying the macro “ReadSwitchStatus()” in the file HardwareProfile_PIC32MX_PIM_Explorer_16.h.

 

·         On USB HID bootloader:

o   Remove the existing linker script file from “USB_HID_Btl_Explorer16”  workspace and add the example linker script file from the location “Bootloader\linker_scripts\PIC32MX_1XX_2XX\example.ld”

o   In the linker script file, modify the line OPTIONAL("libmchp_peripheral_32MX220F032D.a"), to include the correct peripheral library for the chosen part number.

o   This demo can run on PIC32MX2xx PIM + Explorer-16 + USB PICtail board. However, since the switches and LEDs on Explorer-16 board is not mapped to PIC32 device following limitation applies.

1.    The bootloader entry mechanism is not based on switch position (during startup). The bootloader enters firmware upgrade mode only once in the beginning when there is no user application in the Flash.

2.    Once the application is successfully programmed, the bootloader never re-enters the firmware upgrade mode. However, it is recommended to design a re-entry mechanism based on the end product.

o   Note that, the resistors R17 and R18 on the PIM must be removed. The jumpers must be closed on PGED1 and PGEC1 side on the PIM.

 

·         On USB MSD (Thumbdrive) bootloader:

o   Remove the existing linker script file from “USB_MSD_Btl_Explorer16” workspace and add the example linker script file from the location “Bootloader\linker_scripts\PIC32MX_1XX_2XX\example.ld”

o   In the linker script file, modify the line OPTIONAL("libmchp_peripheral_32MX220F032D.a"), to include the correct peripheral library for the chosen part number.

o   User has to design a mechanism suitable to his end product to make the bootloader enter firmware upgrade mode.  This can be done by modifying the macro “ReadSwitchStatus()” in the file HardwareProfile_PIC32MX_PIM_Explorer_16.h.

o   This demo can run on PIC32MX2xx PIM + Explorer-16 + USB PICtail board. However, since the switches and LEDs on Explorer-16 board is not mapped to PIC32 device following limitation applies.

1.    The bootloader entry mechanism is not based on switch position (during startup). The bootloader enters firmware upgrade mode only once in the beginning when there is no user application in the Flash.

Once the application is successfully programmed, the bootloader never re-enters the firmware upgrade mode. However, it is recommended to design a re-entry mechanism based on the end product.

2.    There is no LED indication to indicate the completion of programming. You will have to depend on the thumb drive LED (if applicable) to know the completion of the programming activity.

o   Note that, the resistors R17 and R18 on the PIM must be removed. The jumpers must be closed on PGED1 and PGEC1 side on the PIM.

 

 

 

Demo application support

The demo application code is not directly supported on PIC32MX1xx and PIC32MX2xx devices. User will have to make the following modifications to make the application work on PIC32MX1xx and PIC32MX2xx devices.

·         Remove the existing linker script file from the workspace and add the linker script file “app_pic32mx1xx_2xx.ld”.

·         In the linker script file, modify the line OPTIONAL("libmchp_peripheral_32MX220F032D.a"), to include the correct peripheral library for the chosen part number.

The demo application toggles the I/O port RB8 (Accessible as TP44 on PIC32MX220F032D PIM).  The demo application does not blink any LED on the development board.
